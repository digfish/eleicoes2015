ELEIÇÕES 2015 - API

 Este documento on-line descreve sucintamente a API REST do eleicoes2015.my.to, acompanhada com links para exemplos e respectivo output.

NOTÍCIA

GET /noticia

parâmetros possíveis:

  • vazio (sem parâmetros) – devolve a listagem de todas as notícias

  • term = xx – pesquisa pela ocorrência de xx no título de uma notícia

Devolve (em JSON por omissão) as notícias.

http://eleicoes2015.my.to/api/noticia

POST noticia

Insere uma nova notícia. Os valores dos campos da notícia são enviados sob a forma de um objecto em JSON:

{
    "fonte": "Teste",
    "titulo": "Cavaco Silva anucia finalmente a data das eleições",
    "data": "2015-05-05 13:32:54",
    "final_url": "http://asminhasnoticias.com/"
}

Nota: não se deve especificar qualquer campo “id” nos campos do array JSON, pois uma vez que o registo é completamente novo, tal resultará num erro.

Neste caso a notícia é inserida com sucesso se a resposta contiver o novo id da notícia inserida:

{
    "fonte": "Teste",
    "titulo": "Cavaco Silva anucia finalmente a data das elei\u00e7\u00f5es",
    "data": "2015-05-05 13:32:54",
    "final_url": "http:\/\/asminhasnoticias.com\/",
    "id": 162
}

PUT noticia/:id

Actualiza a informação relativa a uma notícia já presente no sistema. Neste caso deve-se acrescentar o ID da notícia no URL que se pretende actualizar:

Toda a nova informação relativa à notícia deve seguir em formato JSON no corpo da mensagem

Exemplo: PUT http://eleicoes2015.my.to/api/noticia/162

{
    "fonte": "Teste",
    "titulo": "Cavaco Silva enganou-se na data das eleições!",
    "data": "2015-05-07 13:32:54",
    "final_url": "http://asminhasnoticias.com/"
}
A resposta conterá o ID correcto em caso de êxito:
{
    "id": 162,
    "fonte": "Teste",
    "titulo": "Cavaco Silva enganou-se na data das elei\u00e7\u00f5es",
    "original_url": null,
    "data": "2015-05-07 13:32:54",
    "final_url": "http:\/\/asminhasnoticias.com\/"
}

DELETE noticia/:id

Para apagar uma notícia usa-se o comando DELETE da seguinte forma, por exemplo:

DELETE http://eleicoes2015.my.to/api/noticia/162

Em caso de sucesso, o registo na BD apagado é devolvido:

{
    "id": 162,
    "fonte": "Teste",
    "titulo": "Cavaco Silva enganou-se na data das elei\u00e7\u00f5es",
    "original_url": null,
    "data": "2015-05-07 13:32:54",
    "final_url": "http:\/\/asminhasnoticias.com\/"
}

Se fizermos um GET http://eleicoes2015.my.to/api/noticia/162 a resposta será:

{

}

O que prova que a notícia foi correctamente apagada !

POST partido

Cria um novo partido no sistema. À semelhança do caso das notícias deve-se inserir todos os campos com a excepção do campo ID.

TAGS DA NOTICIA

GET noticia/:noticia_id/tag

Devolve todas as tags associadas à notícia de ID :noticia_id .

http://eleicoes2015.my.to/api/noticia/50/tag

[
  {
    "id": 3,
    "nome": "António Costa"
  },
  {
    "id": 26,
    "nome": "Pedro Passos Coelho"
  },

GET noticia/:id/tag/:tag_id

Devolve apenas a tag de ID :tag_id relativa à notícia de tag :noticia_id.

Aproveitando o exemplo, suponhamos que queremos apenas a tag de id 26:

http://eleicoes2015.my.to/api/noticia/50/tag/26

[
  {
    "noticias_id": 50,
    "tags_id": 26,
    "id": 26,
    "nome": "Pedro Passos Coelho"
  }
]

DELETE noticia/:noticia_id/tag/:tag_id

Elimina a associação da noticia :noticia_id com a tag :tag_id .

Por exemplo, se quisermos eliminar a associação da notícia 1000 com as tags de IDs 1000 e 1001 efectuadas pelo comando anterior “POST” é necessário

fazer duas invocações ao comando:

DELETE http://eleicoes2015.my.to/api/noticia/100/tag/1000

e

DELETE http://eleicoes2015.my.to/api/noticia/100/tag/1001



TAGS

GET tag/

Lista todas as tags registadas no sistema. Os parâmetros que se aplicam ao recurso “noticia” também se aplicam aqui.

Entre os campos devolvidos, inclui-se a ocorrência relativa a cada tag. A lista aparece ordenada pelo número de ocorrências de cada tag.

http://eleicoes2015.my.to/api/tag/

[
    {
        "count": 29,
        "nome": "António Costa",
        "id": 3
    },
    {
        "count": 24,
        "nome": "União Europeia",
        "id": 65
    },
    {
        "count": 19,
        "nome": "Pedro Passos Coelho",
        "id": 26
    },

GET tag/:id

Devolve apenas a tag de ID :id, incluindo o seu número de ocorrências.

Exemplo: mostrar a tag de ID 3 (“António Costa”)

http://eleicoes2015.my.to/api/tag/3

[
    {
        "count": 29,
        "nome": "António Costa",
        "id": 3
    }
]

O parâmetro “count” refere que esta tag ocorre em 29 notícias.

POST tag

Insere uma nova tag no sistema.

Exemplo:

Invocando http://eleicoes2015.my.to/api/tag/ com a seguinte mensagem JSON

{
 "nome": "MPT"
}

irá despoletar a criação da tag no sistema, devolvendo:

    {
        "count": 0,
        "nome": "MPT",
        "id": 5000
    },

PUT tag/:id

Actualiza a palavra da tag :id com uma nova palavra.

PUT http://eleicoes2015.my.to/api/tag/3

{
 "nome": "Rui Costa"
}

Irá actuializar o texto da tag “António Costa” com “Rui Costa”

DELETE tag/:id

Apaga a tag de ID :id.

Exemplo:

PUT http://eleicoes2015.my.to/api/tag/3

Irá apagar a tag de ID 3.
